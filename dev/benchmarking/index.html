Effect of logging $R_t$

Logging $R_t$ in each timestep slows the simulation down by a lot.

using Daedalus
using BenchmarkTools

# turn off Rt logging and compare with benchmark above
@benchmark daedalus(country="Australia", log_rt=false, time_end=600.0)

Reactive events

Reactive events currently require logging $R_t$ so are not benchmarked with logging turned off.

using Daedalus
using BenchmarkTools

npi = Daedalus.DaedalusStructs.Npi(20000.0, (coef=0.7,));

@benchmark daedalus(country="Australia", r0=5.0, npi=npi, time_end=600.0)

Timed events

# benchmark for a full run of 600 days
using Daedalus
using BenchmarkTools

timed_npi = Daedalus.DaedalusStructs.TimedNpi(
    [60.0, 180.0, 300.0],
    [120.0, 200.0, 365.0],
    [0.3, 0.7, 0.5],
    "three_phase_lockdown"
)

@benchmark daedalus(country="Australia", r0=3.0, npi=timed_npi, time_end=600.0)
